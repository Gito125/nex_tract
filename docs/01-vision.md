# 01 — Nextract Project Vision

## 1. Project Name

**Nextract**

The name comes from:

```txt
Nexus + Extract = Nextract
```

### Meaning

Nextract is the point where different media platforms connect into one clean personal extraction tool.

- **Nexus** represents connection, centralization, and a hub.
- **Extract** represents pulling media out into a usable personal archive.

The name feels short, modern, technical, and brandable.

## 2. Project Summary

Nextract is a clean, simple, and beautiful media downloader application.

It allows users to paste a media link, preview the media, choose the quality, and download the content into an organized local library.

The app begins with YouTube support, including:

- Single video downloads.
- Different quality options.
- Audio-only downloads.
- Full playlist downloads.
- Progress tracking.
- Download history.

Later, it can expand to support major social media platforms such as:

- TikTok.
- Instagram.
- X.com.
- Facebook.
- Reddit.
- Vimeo.
- SoundCloud.
- Other supported media platforms.

## 3. Product Positioning

Nextract is not a piracy tool.

Nextract is a **personal media extraction and archiving tool** for:

- Content the user owns.
- Content the user created.
- Content the user has permission to download.
- Public domain or freely licensed media.
- Educational/offline personal use where allowed.
- Creator backups.
- Research/reference archiving.

Nextract should never be marketed as a tool to bypass restrictions, download paid content, scrape private accounts, or break platform rules.

## 4. Problem Statement

Many people want to save useful media for offline use, but existing downloader tools are often:

- Ugly.
- Overloaded.
- Too technical.
- Untrustworthy-looking.
- Full of ads.
- Inconsistent.
- Unsafe-feeling.
- Poorly organized.

Nextract solves this by giving users a clean, polished, and controlled media downloading experience.

The goal is not just to download files.

The goal is to make media extraction feel organized, safe, and professional.

## 5. Core Vision

The user should feel like they are using a beautiful personal media vault.

The main experience should be:

```txt
Paste link → Analyze media → Choose quality → Download → Organized file saved
```

The app should hide technical complexity while still allowing advanced users to access deeper options when needed.

## 6. Target Users

### Primary Users

- Students saving educational content for offline study.
- Creators backing up their own videos.
- Developers who want a clean GUI over yt-dlp.
- Researchers archiving public media references.
- Users with unstable internet access.
- Teachers collecting offline learning materials.

### Secondary Users

- Podcasters saving audio references.
- Editors collecting media assets they are allowed to use.
- Content managers organizing public media resources.
- Personal users building offline libraries.

## 7. Product Personality

Nextract should feel:

- Clean.
- Calm.
- Modern.
- Fast.
- Reliable.
- Beautiful.
- Minimal.
- Trustworthy.
- Professional.
- Lightweight.

It should not feel:

- Sketchy.
- Noisy.
- Ad-heavy.
- Illegal.
- Confusing.
- Over-engineered.
- Like a random downloader site.

## 8. Design Philosophy

The design should be simple first.

The home screen should not overwhelm the user with settings.

The most important action is:

```txt
Paste a link
```

Everything else should appear after the link is analyzed.

Advanced options should be hidden but accessible.

## 9. Visual Direction

### Layout Style

- Card-based interface.
- Soft rounded corners.
- Large paste-link area.
- Clear call-to-action button.
- Spacious layout.
- Clean navigation.
- Simple icons.
- Smooth progress bars.
- Friendly empty states.

### Color Direction

Recommended visual direction:

- Light background: soft white or light gray.
- Primary color: deep indigo, blue, or violet.
- Accent color: cyan, blue, or soft green.
- Dark mode: deep navy, charcoal, or near-black.
- Success: calm green.
- Warning: amber.
- Error: soft red.

### Typography

Typography should feel modern and readable.

Recommended style:

- Strong headings.
- Clean body font.
- Clear labels.
- No decorative overload.

## 10. Main Screens

### 1. Home Screen

Purpose: Let the user paste a link and start.

Must include:

- App name: Nextract.
- Hero text.
- Link input box.
- Analyze button.
- Supported platform icons.
- Recent downloads preview.
- Clean empty state.

Suggested hero text:

```txt
Extract allowed media into your personal vault.
```

Suggested subtext:

```txt
Paste a link, choose your quality, and save videos, audio, and playlists in one clean space.
```

### 2. Media Preview Screen

Purpose: Show analyzed media before downloading.

Must include:

- Thumbnail.
- Title.
- Platform badge.
- Duration.
- Creator/channel if available.
- Quality selector.
- Audio-only selector.
- Download button.
- Advanced options toggle.

### 3. Playlist Screen

Purpose: Let the user download a full playlist or selected items.

Must include:

- Playlist title.
- Thumbnail/cover.
- Total number of items.
- Select all.
- Select specific videos.
- Download range.
- Skip existing option.
- Quality selection.
- Playlist folder setting.

### 4. Download Queue Screen

Purpose: Track active downloads.

Must include:

- Active jobs.
- Progress bar.
- Percentage.
- Speed if available.
- ETA if available.
- Current status.
- Pause/cancel/retry actions.
- Completed state.
- Failed state.

### 5. History Screen

Purpose: Show previously downloaded media.

Must include:

- Search.
- Platform filters.
- Date filters.
- Download status.
- File path.
- Open file/folder action.
- Re-download action.

### 6. Settings Screen

Purpose: Control app behavior.

Must include:

- Download folder.
- Default quality.
- Default audio format.
- Theme.
- Filename template.
- Playlist behavior.
- Max parallel downloads.
- Save thumbnails.
- Save subtitles.
- Advanced tools path.

## 11. Long-Term Vision

Nextract can grow from a local downloader into a complete personal media management tool.

Possible future features:

- Desktop app.
- Browser extension.
- Batch link import.
- Creator backup mode.
- Subtitle manager.
- Metadata export.
- Scheduled downloads.
- Local media library.
- Smart organization.
- Duplicate detection.
- Download reports.

## 12. Success Definition

Nextract is successful when a user can open it, paste a link, choose quality, and download media without thinking about commands, formats, or technical details.

The experience should feel so clean that the user forgets yt-dlp is running behind the scenes.
